## Findings considered and rejected (do not re-audit)

- **`signals.py` `["request_id"]` KeyError risk** — refuted: django-structlog
  binds `request_id` in `RequestMiddleware` before either subscribed signal
  can fire (verified in the installed middleware source); no other sender
  exists.
- **Unparenthesized `except A, B, C:` in `routes/ready.py:44`** — valid PEP
  758 syntax on the pinned Python 3.14; confirmed by parse and by the passing
  suite. Readability-only concern; deliberately kept.
- **"Missing DJANGO_SETTINGS_MODULE for gunicorn/celery"** — refuted
  empirically: `src/config/__init__.py:3` sets it for every process importing
  the `config` package.
- **"Gunicorn strips X-Forwarded-Proto from untrusted proxies → redirect
  loop"** — corrected by reading gunicorn 26 source: headers pass through to
  the WSGI environ; the real issue is spoofability + missing proxy contract,
  handled in plan 002.
- **Settings overlays' `noqa: F821` pattern (7 sites)** — inherent to
  django-split-settings' exec-into-namespace model; alternatives contradict
  the documented architecture (django-configurations is forbidden by
  AGENTS.md). Keep.
- **django-cors-headers** — explicitly rejected by the maintainer.
- **API throttling/rate limiting** — explicitly rejected by the maintainer.
- **django-celery-email for async mail** — rejected: last release Dec 2019,
  pre-Celery 5, unmaintained; plan 009 uses a project-owned task instead.
- **`requires-python = ">=3.14,<3.15"` upper bound** — reviewed; left as a
  maintainer policy choice (single-minor support window is deliberate).
- **Admin hardening as its own plan** — folded into plan 002's Production
  README notes (proxy-level restriction); stock-Django exposure otherwise.
- **`.editorconfig`** — low value: Ruff-format/yamlfmt/whitespace hooks
  already enforce formatting.
- **Dedicated real-backend pytest lane (Postgres/Redis service containers)**
  — superseded by plan 013's smoke test, which exercises `/api/ready` against
  real backends in the booted stack.
- **Abstract `apps.core` models with no concrete users** — intended
  scaffolding; standard low-cost mixins. Keep.
- **DEBUG re-assignments in `ci.py`/`prod.py`** (already `False` in
  `core.py`) — harmless explicitness; not worth a change.
