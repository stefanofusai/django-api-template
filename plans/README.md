## Findings considered and rejected

Recorded so nobody re-audits them. (Earlier cycles' rejection lists:
`git show e0ec725^:plans/README.md`.)

Rejected by the cycle-3 audit (2026-07-09, at `e0ec725`):

- **"Unreachable 403 responses" in `notes/controllers.py`**: reachable in
  half the knob space — 403 is declared only on the mutating endpoints,
  exactly where session-mode CSRF failures return 403 (`django_auth` is
  ninja's `SessionAuth(csrf=True)`). Dead only in the JWT branch; making the
  declaration Jinja-conditional would churn the committed OpenAPI schemas
  for cosmetics. Not worth doing.
- **JWT `SIGNING_KEY` reuses `SECRET_KEY`**: the django-ninja-jwt default
  made explicit; standard symmetric-HS256 practice. Not a finding.
- **"Docker image only build-tested with default knobs"**: overstated — the
  smoke matrix builds the image for three variants covering most Dockerfile
  Jinja branches. Residual gap: the `email_provider=smtp` branch
  (`Dockerfile:51`) is hadolint-linted but never `docker build`-executed.
  Accepted as marginal; revisit only if an smtp-branch build bug actually
  ships.
- **Unguarded concurrent migrations on multi-replica scale-out**: outside
  the template's documented single-host Compose deployment model.
- **`ZEAL_ALLOWLIST` duplicated across dev/ci overlays**: deliberate,
  documented duplication (comments in both files say "kept in sync");
  consistent with previously rejected micro-consolidations.
- **Schemathesis excludes `/api/v1/token/*` from the contract gate**:
  deliberate (library-owned operations). Plan 004 adds the explanatory
  comment; the exclusion itself stays.
- **CSP applies `unsafe-eval`/`style-src unsafe-inline` globally**:
  by-design for self-hosted Swagger + unfold (the primary XSS vector —
  inline `script-src` — remains blocked); nonce-based CSP is a recorded dead
  end (see cycle-2 notes). Possible future hardening: per-view header
  scoping. Not a defect.
- **Backup `verify` empty-archive check (`-s`) can't trigger for gzip
  output**: a gzip stream is never 0 bytes, so the guard is weak — but the
  docker-run exit status under `set -eu` already catches real failures.
  Harmless belt-and-suspenders; not worth changing.
