`{{cookiecutter.project_slug}}/` (quote it in shell), files inside it
contain Jinja placeholders that must stay valid, and verification always
means **baking** a project (`uvx cookiecutter . --no-input -o <dir>`)

inside the template are copied WITHOUT rendering — never put Jinja or
slug-derived values there. After plan 001 lands, running the baked suite
requires a reachable Postgres (see that plan's README updates).

- **No image tagging/registry → no fast rollback** (prod compose builds
  in place; `docker-build.yaml` uses `push: false`). Real, MED effort;
  needs a maintainer call on registry/tagging conventions. Deferred.
- **No metrics surface** (no `/metrics`, no Flower; Sentry-only
  observability). Deliberate scope so far; revisit if operating pain
  appears. The resource bounds in plan 007 are the cheap subset.
- **No CSP/Permissions-Policy on the admin surface** — LOW confidence
  value; admin exposure is already documented as a proxy-level concern.
  Recorded, deferred.
- **Release/versioning workflow** (DIR: `version = "1.0.0"` is static
  but feeds Sentry release tagging, so releases never move without a
  manual bump). Grounded; needs a tooling decision (release-please vs
  tag-driven). Deferred to a maintainer decision.
- **cruft template-update story** — deferred again (recorded twice now);
  the gap between the template and already-baked projects keeps growing.
- **`justfile` task runner** for the long compose command lines — offer
  stands; philosophy headwind (tooling minimalism), maintainer's call.
- **API auth knob (token/JWT)** — partially addressed by plan 009's
  `django_auth` example; a dependency-adding auth knob remains deferred.
- **Shipped `deploy.resources` memory limits** — documented-only in plan
  007; wrong defaults OOM-kill legitimate workloads.

## Settled decisions (carried from the retired plan set — do not re-audit)

- **django-cors-headers**: rejected by the maintainer.
- **API throttling/rate limiting**: rejected by the maintainer.
- **PgBouncer in prod.yaml**: evaluated and rejected (steady-state ~8-9
  connections vs max 100; transaction pooling would force
  `CONN_MAX_AGE=0` and break session semantics). Escalation path:
  psycopg3 native pool first.
- **`.editorconfig`**: rejected, low value next to existing hooks.
- **django-celery-email**: rejected (unmaintained); the project-owned
  `send_email` task is the pattern.
- **`requires-python = ">=3.14,<3.15"` upper bound**: deliberate
  single-minor support window.
- **Unparenthesized `except A, B, C:` in `routes/ready.py`**: valid PEP
  758 on Python 3.14; deliberately kept.
- **`noqa: F821` pattern in settings overlays**: inherent to
  django-split-settings; keep. django-configurations is forbidden.
- **`signals.py` request_id KeyError**: refuted (django-structlog binds
  it before the signals fire).
- **Admin hardening**: proxy-level restriction, documented in the baked
  README; not an app-level plan.
- **Abstract `apps.core` models with no concrete users**: intended
  scaffolding (plan 009 finally consumes them).
- **Redis `noeviction`**: documented tradeoff (broker durability); plan
  007 adds the missing memory *bound*, not a policy change.
- **Traefik + docker-rollout as shipped default**: maintainer decision
  2026-07-04 (reversed the earlier bring-your-own-ingress stance).
- **Sentry boot-required in prod when `use_sentry=yes`**: deliberate.
- **Knob defaults reproduce the historical full-stack output**
  (byte-for-byte) — every new knob must honor this (see plan 009 Step 8).
- **SQLite-in-tests**: the old audit's "real-backend pytest lane
  superseded by the compose smoke test" verdict is **overridden** by an
  explicit 2026-07-05 maintainer mandate — plan 001.
