## Findings considered and rejected

- **JWT without the notes example is a silent bug**: rejected. The root
  variable table explicitly says `api_auth` only takes effect when
  `use_example_api=yes`. Plan 025 treats decoupling as a direction decision.
- **JWT token-pair is currently unprotected by Axes**: rejected as a bug after
  verifying the pinned library path; plan 009 adds a regression test.
- **Traefik retries POST/PATCH writes by default**: rejected. Pinned Traefik
  defaults `retryNonIdempotentMethod` to false.
- **Pin every GitHub Action by commit SHA**: not planned because
  `.github/zizmor.yaml` explicitly chooses Dependabot-managed version refs.
- **Hash every vendored skill reference file**: rejected because the previous
  accepted design intentionally defines `computedHash` as raw `SKILL.md`
  bytes. Revisit only if the lock schema changes.
- **Add trigram indexes to notes search**: not worth the migration/extension
  cost for the deliberately small exemplar. Plan 013 fixes correctness only.
- **Boot-test external Traefik certificates**: accepted residual gap; real
  certificate setup outweighs current regression value.
- **Pass Redis password through a mounted config instead of command args**:
  rejected for now. Docker operators who can inspect process args can already
  inspect container environment; proposed alternatives add secret-file/race
  complexity without changing that trust boundary.
- **General inverse guard for every new Jinja-conditional file**: rejected as
  under-specified. Existing removal entries are fail-closed and existence-
  tested; no reliable add-side inventory can be inferred without introducing a
  new manifest contract. Plan 015/019 add targeted truth-table invariants.
- **Deduplicate root/generated zizmor or Compose service blocks**: rejected;
  they serve different repositories/topologies and abstraction cost exceeds
  current drift risk.
- **Add `.editorconfig`**: rejected again; existing format/EOL hooks cover the
  concrete problem.
- **Document Django 6.0 EOL as a static date**: rejected in favor of automated
  dependency freshness (plan 017), which does not create stale lifecycle prose.
- **Add a staging overlay immediately**: low-confidence; plan 027 is a demand
  spike and defaults to defer.
